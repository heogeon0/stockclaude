## 작성 원칙

1. **매매 이력은 테이블 필수** — 시간순 누적, 실현이익 컬럼 유지
2. **감시 레벨 가격은 구체 수치만** — "%"만 쓰면 안 됨, ₩XXX,XXX 명시
3. **주식수도 구체 수치** — "절반" X, "10주" O
4. **갱신일 명시** — 현재가/평가손익은 언제 기준인지 표시
5. **상태 뱃지** — Active 기본, 청산 시 Close + Close 파일은 매매 이력 보존
6. **변동성 등급 + 재무 등급은 매일 daily 시 자동 갱신** (`analyze_volatility` + `compute_financials` 결과 기반)
