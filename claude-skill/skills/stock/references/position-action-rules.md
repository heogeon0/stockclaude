# 포지션 반영 룰 (보유 종목 액션 결정)

> 로드 시점: 보유 종목 분석 시 (position 데이터 + 시그널 조합).

## 6대 룰

### 1. 수익률 계산 (자동)

```
수익률 = (현재가 - 평단가) / 평단가 × 100
```

`compute_indicators(code)` 의 `price_context` + `get_portfolio.positions[code].avg_price` 기반.

### 2. 보유 + 매도 시그널 다수 → "이익 실현 or 손절 검토"

```
종합 verdict = 매도우세 / 강한매도
AND 수익률 > +10% → 부분 익절 검토 (변동성×재무 셀의 피라미딩 단계 = 익절 단계)
AND 수익률 -5% ~ +5% → 손익분기 구간 (트레일링 스탑 점검)
AND 수익률 < -5% → 손절 검토
```

### 3. 보유 + 매수 시그널 다수 → "피라미딩 검토" (조건부)

```
종합 verdict = 강한매수 / 매수우세
AND 비중 < 25% (집중도 룰)
AND 변동성×재무 셀의 피라미딩 단계 > 0
AND check_concentration() 통과
THEN 피라미딩 검토
```

**단, `check_concentration` 먼저** — 집중도 25% 룰 위반 시 자동 차단.

### 4. 미보유 + 매수 시그널 다수 → "신규 진입 + 재무 확증"

```
종합 verdict = 강한매수 / 매수우세
AND 변동성×재무 셀 = 비추 아님
AND 진입 사이즈 (cell.size) 적용
THEN 신규 진입 후보 (단, base.md 작성 또는 컨센 확인 필수)
```

### 5. 손절선 -3% 이내 → 최상단 ⚠️ 경고

```
손절가까지 거리 = (현재가 - 손절가) / 현재가 × 100
IF abs(손절가까지 거리) < 3% THEN
  daily 보고서 최상단 ⚠️ 경고 박스
  "손절 -3% 이내 — 즉시 결정 필요"
```

### 6. 평단 ±2% → "손익분기 구간"

```
IF abs(수익률) < 2% THEN
  "손익분기 구간 — 트레일링 스탑 점검"
  → 추가 매수 / 매도 모두 신중
```

## 수익률 구간별 액션 매트릭스

| 수익률 | 시그널 매수우세 | 시그널 중립 | 시그널 매도우세 |
|---|---|---|---|
| **+30%+** | 트레일링 스탑 타이트화 | **부분 익절 (1/3)** | **즉시 부분 익절 (1/2)** |
| **+15~30%** | 홀딩 | 트레일링 스탑 | 부분 익절 (1/3) |
| **+5~15%** | 피라미딩 검토 | 홀딩 | 트레일링 스탑 점검 |
| **-5~+5%** | 1차 진입 검토 | 손익분기 — 결정 보류 | **본전컷 검토** |
| **-5~-10%** | 추가 진입 신중 | 손절 검토 | **즉시 손절** |
| **-10%+** | ❌ 추가 매수 금지 | **즉시 손절** | **즉시 손절** |

## 손절 단계 (변동성×재무 셀의 손절폭 적용)

각 셀별 단계 손절:

```
경고선: 평단 -50% × 손절폭 (절반 지점 — 관망)
기준선: 평단 -75% × 손절폭 (3/4 지점 — 부분 손절 1/3)
손절선: 평단 -100% × 손절폭 (잔여 전량)
```

예: A급 normal 셀 (-10% 손절폭) →
- 경고선 -5% (관망)
- 기준선 -7.5% (1/3 매도)
- 손절선 -10% (잔여 전량)

ATR 기반 셀 (extreme 변동성) →
- ATR×1 (관망)
- ATR×1.5 (1/3 매도)
- ATR×2 (잔여 전량)

## 매매 집행 전 체크리스트

매수/피라미딩/매도 결정 직전:

```
[ ] check_concentration → 25% 룰 통과
[ ] 예수금 충분 (currency 기준)
[ ] 실적 D-7 이내면 손절선 타이트화
[ ] 최근 외인/기관 수급 톤 (analyze_flow)
[ ] 동일 섹터 보유 총합 (portfolio_correlation)
[ ] W17 회고 룰 (rule_win_rates) — 승률 < 50% 룰은 추가 검증
[ ] 변동성×재무 셀 사이즈 적용
[ ] 집행 후 record_trade 호출 (트리거가 positions 자동 재계산)
[ ] 필요 시 propose_watch_levels(persist=True) 로 감시 레벨 자동 생성
```

## 분할 익절 룰 (W17 강화)

전량 익절 ❌ — 단계별 매도 ✅:

```
1차 목표 도달 → 1/3 매도
2차 목표 도달 → 추가 1/3 매도
3차 목표 또는 트레일링 스탑까지 → 잔여 1/3 보유
```

이유: W17 회고 "효성 4/23 익절 후 +₩330k 추가 상승 놓침" 사례.

→ 변동성×재무 셀의 피라미딩 단계 = 익절 단계 (대칭).
