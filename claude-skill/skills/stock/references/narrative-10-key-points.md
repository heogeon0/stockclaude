# 10 Key Points + So What 구성 룰

> 로드 시점: base.md 최상단 IC 메모 작성 시.

## 10 Key Points 구성 (반드시 10개, 한 줄씩)

| # | 항목 | 내용 |
|---|---|---|
| ① | **최종 판단** | 종합 결론 한 줄 (보유/추가매수/관망/매도) |
| ② | **Narrative 정의** | 시장이 가격에 반영한 스토리 (한 줄) |
| ③ | **Reverse DCF 인사이트** | 현 주가가 암시하는 성장·마진·재투자 |
| ④ | **Narrative 현실성 검증** | 산업 구조상 가능한가 — 통계·외부 데이터 근거 |
| ⑤ | **DCF 인사이트** | 나의 가정 하 적정가 (확률가중) |
| ⑥ | **Comps 인사이트** | 피어 대비 프리미엄/디스카운트 (백분위) |
| ⑦ | **가장 중요한 변수** | 밸류에이션 좌우 변수 1개 (단 하나) |
| ⑧ | **시장이 놓치고 있는 것** | 과소/과대평가 요인 (Edge) |
| ⑨ | **최대 리스크 + 딜 레이더** | 구조적 리스크 + 진행 중 딜 |
| ⑩ | **업사이드 촉매 + 액션** | 상승 트리거 + 다음 확인 항목 |

## 작성 가이드

### 길이
- 각 항목 **1줄 (50~80자)**
- 길어지면 본문 섹션으로 이동

### 톤
- 직접·단정적으로 (예: "X 회복 확률 60% 미만이면 비싸다")
- 모호한 표현 금지 (예: "양호한 성장세" → "매출 CAGR +12%")

### 데이터 인용
- 모든 숫자는 **본문 섹션 근거** 필수
- 본문에 없는 숫자는 10 Key Points 사용 금지

## So What — 투자 판단 요약

10 Key Points 직후 **So What** 섹션으로 압축:

```markdown
## 💡 So What — 투자 판단 요약

### 확률가중 적정가
- Bull X% × ₩Y = ₩Z
- Base X% × ₩Y = ₩Z
- Bear X% × ₩Y = ₩Z
→ 확률가중: ₩X
→ 현 주가 대비: ±X% 업사이드

### 한 줄 판단
"현 주가는 [시나리오]가 [X]%+ 실현 필요. [핵심변수] 성공 확률 [X]% 이하면 비싸다."

### 이벤트별 주가 영향
- [이벤트1] → ±X% (±₩Y/주)
- [이벤트2] → ±X%
- [이벤트3] → ±X%

### 보유 현황 (position 있을 때)
- 평단/수익률/판단(추가매수 / 보유 / 분할매도 / 손절검토)
```

## 핵심 변수 정의

10 Key Points ⑦ 의 "가장 중요한 변수" 는 본문 별도 섹션으로 확장:

```markdown
## 핵심 변수 (밸류에이션 좌우)

1. [변수1] — 민감도: 1%p 변화 시 주가 ±X%
2. [변수2] — 민감도
3. [변수3] — 민감도
```

각 변수는:
- **수치로 표현 가능해야 함** (예: "HBM 점유율 %", "1Q26 영업이익률 %")
- **모니터링 가능해야 함** (분기 실적 / 공시 / 컨센)
- **Bull/Base/Bear 시나리오 분기점**과 명확히 매칭

## 길이 가이드

- 10 Key Points: 약 1페이지
- So What: 약 1페이지
- 둘 합쳐 base.md 의 처음 2~3페이지 차지 — IC 미팅 1차 자료
