# 룰 카탈로그

> `record_trade` 시 `rule_category` enum / `weekly_reviews.win_rate` jsonb 키 정의처.
> 매매 시점에 어떤 룰을 적용했는지 명시 → 매주 회고에서 룰별 win/loss 자동 그룹.
> 룰별 누적 승률 → daily 매매 시 자동 경고 (예: "신고가돌파매수 33% 승률 → D+1 안착 룰 강제").

---

## 핵심 15 룰 (Phase 1 — 사용자 trades 패턴 + 시스템 도구 매핑)

### 🟢 진입 룰 (6)

#### 1. `강매수시그널진입`
- **트리거**: `compute_signals(code).summary` 의 매수 ≥ 3 & 매도 = 0 ("강한매수")
- **조건**: 변동성×재무 셀이 비추 X (D-extreme 제외)
- **회피**: 실적 D-7 이내 / 컨센 -15% 하향
- **win 정의**: 진입 후 20일 내 +5% 이상 도달
- **사이즈**: 매트릭스 셀 풀 사이즈

#### 2. `신고가돌파매수`
- **트리거**: 52주 신고가 갱신 + 거래량 1.5x 이상 + `detect_events.price_break = "high"`
- **조건**: D+1 안착 룰 강제 (W17 회고 — 33% 승률 발견)
- **회피**: RSI 80+ 동시 발생
- **win 정의**: 진입 후 D+1 종가 평단 +/-2% 내 안착
- **사이즈**: 매트릭스 셀 70% (과열 위험 헤지)

#### 3. `가치신규진입`
- **트리거**: PBR < 1.0 + ROE 양호 + `compute_signals` 강매수 + `compute_score.financial ≥ B`
- **조건**: 변동성 normal/high (extreme 제외)
- **회피**: 산업 사이클 침체
- **win 정의**: 진입 후 30일 내 +10% 또는 PBR 1.0 도달
- **사이즈**: 매트릭스 셀 풀

#### 4. `VCP_SEPA진입`
- **트리거**: `chart_analysis` 의 VCP 수축률 < 10% + SEPA 정석 패턴
- **조건**: 거래량 동반 + 모멘텀 A/B 등급
- **회피**: 시장 regime bear
- **win 정의**: 진입 후 20일 내 피봇 +8% 돌파
- **사이즈**: 매트릭스 셀 70% (정밀 진입)

#### 5. `피라미딩D1안착`
- **트리거**: 직전 신고가돌파매수 trade 의 D+1 종가가 평단 ±2% 내 안착 + 거래량 동반
- **조건**: 모멘텀 가동 regime
- **회피**: D+1 -3%+ 반락 시 즉시 컷
- **win 정의**: 피라미딩 후 10일 내 +5%
- **사이즈**: 1차 진입의 30% (보조)

#### 6. `실적D-1선제진입`
- **트리거**: `detect_events.earnings_dte ≤ 2` + 강매수 시그널 + 컨센 평균 목표가 ≥ 현재가 +10%
- **조건**: **반드시 `이벤트익절` (sell) 과 양면 베팅** — 모멘텀 잡되 변동성 헤지로 비중 축소
- **회피**: 컨센 ±5% 내 (서프라이즈/쇼크 양방향 큼) / 변동성 extreme regime
- **win 정의**: 실적 발표 후 D+1 종가가 진입가 +5%+
- **사이즈**: 1차 진입의 20~30% (위험 큼, 작게)
- **위험 메모**: 양방향 ±5~15% 변동 가능. 단독 진입 X (반드시 이벤트익절 동반)

### 🔴 청산 룰 (6)

#### 6. `1차목표도달익절`
- **트리거**: `propose_watch_levels` 의 `target_1` 가격 도달
- **조건**: 익절 비율 — 매트릭스 셀별 (extreme 50% / high 33% / normal 25%)
- **회피**: 임박 실적 서프라이즈 + 모멘텀 강 (홀딩 가능)
- **win 정의**: 익절가 ≥ 평단 +10%
- **사이즈**: 셀별 부분 익절

#### 7. `이벤트익절`
- **트리거**: 실적 D-1 또는 신고가 도달 시 비중 축소 (변동성 헤지)
- **조건**: `detect_events.earnings_dte ≤ 1` 또는 `price_break = "high"`
- **회피**: Premium 등급 + 강매수 시그널 (홀딩)
- **win 정의**: 익절가 평단 +5% 이상
- **사이즈**: 33% 회수

#### 8. `모멘텀꼴찌청산`
- **트리거**: 포트폴리오 내 `rank_momentum.z_score` 꼴찌 + 기준선 (전환선) 이탈
- **조건**: 포트 분산 / 비중 조정 차원
- **회피**: 단순 일시 조정 (5일 평균 z-score 도 꼴찌일 때만)
- **win 정의**: 청산 후 30일 내 추가 -5% 하락 (회피 성공)
- **사이즈**: 전량

#### 9. `피라미딩실패컷`
- **트리거**: 피라미딩 분 D+2 종가가 진입가 -3%+ 반락
- **조건**: W17 회고 룰 100% 승률 (강제 적용)
- **회피**: 없음 (Always)
- **win 정의**: 피라미딩분만 회수 (1차 분 보존)
- **사이즈**: 피라미딩분 100%

#### 10. `RSI과열청산`
- **트리거**: RSI 90+ + 매도 시그널 ≥ 5건
- **조건**: 평단 +20%+ (과열 동반)
- **회피**: 강한 모멘텀 (그랜빌 SMA60 위 등) — 홀딩
- **win 정의**: 청산 후 10일 내 -5% 반락 (성공)
- **사이즈**: 33~50% 부분

#### 11. `Defensive조기익절`
- **트리거**: `compute_score.grade = "Defensive"` + 평단 +15% 도달
- **조건**: 등급 한계 — 더 이상 보유 효익 X
- **회피**: 컨센 +15% 상향 (홀딩 정당화)
- **win 정의**: 익절가 평단 +15%+
- **사이즈**: 전량 또는 50%

### 🟡 관리 룰 (3)

#### 12. `집중도25%회수`
- **트리거**: `check_concentration` 결과 25% 룰 위반 (단일 종목 / 섹터)
- **조건**: 매수 / 피라미딩 시 자동 차단
- **회피**: 사용자 명시 "무시하고 집행"
- **win 정의**: 회수 후 포트 다각화 점수 향상
- **사이즈**: 25% 초과분만

#### 13. `ATR손절`
- **트리거**: 진입가 - ATR×2 손절선 이탈 (intraday + 종가 둘 다)
- **조건**: 매트릭스 셀별 ATR 배수 (extreme 1.5 / high 2.0 / normal 2.5)
- **회피**: 시장 regime 일시 패닉 (5일 회복 시 재진입 검토)
- **win 정의**: 손절 후 추가 -5% 하락 회피 성공
- **사이즈**: 전량

#### 14. `컨센하향청산`
- **트리거**: 1M 평균 목표가 -15% 하향 (3사+) + 다운그레이드 리포트 ≥ 2건
- **조건**: `analyze_consensus_trend` + `list_analyst_reports` 동시 충족
- **회피**: Premium 등급 + 매수 시그널 강 (재검토)
- **win 정의**: 청산 후 30일 내 -10%+ 추가 하락
- **사이즈**: 50~100%

## 사용 룰 (record_trade 시점)

```python
record_trade(
    code='005930',
    side='buy',
    qty=8,
    price=200_000,
    executed_at='2026-04-28T14:30:00',
    rule_category='가치신규진입',  # 카탈로그 enum
    trigger_note='PBR 0.85 + 1Q26 사상최대 + 강매수 시그널',  # 자유 텍스트 (상세 사유)
    fees=600,
)
```

**rule_category** = 카탈로그 enum (이 14 룰 중 1개)
**trigger_note** = 자유 텍스트 — 상세 사유 / 데이터 / 컨텍스트 (룰 명시 후 보조 설명)

## 룰 추가 절차 (Phase 2+)

1. 새 룰 후보 — 사용자 매매 패턴 또는 시스템 도구 미활용 영역 식별
2. 카탈로그에 정의 추가 (트리거 / 조건 / 회피 / win 정의 / 사이즈)
3. `scripts/XX_rule_category_add.sql` 마이그레이션 — CHECK constraint 갱신
4. SKILL.md 매매 체크리스트 보강
5. weekly_reviews.win_rate jsonb 키 통일

---

## 룰 폐지 기준

- **4주 누적 승률 < 30%** → 폐지 후보 (사용자 검토)
- **12주 누적 승률 < 40%** → 자동 폐지 권장 (회고 시 명시)
- **시스템 도구 변화로 트리거 무의미** → 즉시 폐지

## 룰 카테고리 enum (DB CHECK constraint 후보)

```sql
CHECK (rule_category IN (
    -- 진입 (6)
    '강매수시그널진입', '신고가돌파매수', '가치신규진입', 'VCP_SEPA진입', '피라미딩D1안착', '실적D-1선제진입',
    -- 청산 (6)
    '1차목표도달익절', '이벤트익절', '모멘텀꼴찌청산', '피라미딩실패컷', 'RSI과열청산', 'Defensive조기익절',
    -- 관리 (3)
    '집중도25%회수', 'ATR손절', '컨센하향청산',
    -- 누락 / 옵셔널
    NULL  -- 옛 trades 또는 명시 안 한 매매
))
```

## 활용 흐름

```
[매매 시점]
  record_trade(rule_category='신고가돌파매수', ...)
  → trades.rule_category = '신고가돌파매수' 영구 기록

[매주 회고 시]
  list_trades_by_rule(week_start='2026-04-20')
  → {'신고가돌파매수': [trade1, trade2], '이벤트익절': [...]}
  → 사용자가 룰별 trade 보고 win/loss 평가
  → save_weekly_review(win_rate={'신고가돌파매수': {tries: 3, wins: 1, pct: 33.3}, ...})

[다음 주 daily]
  get_weekly_context(weeks=4)
  → rolling_stats.rule_win_rates: {'신고가돌파매수': 33%}
  → daily 가 신고가돌파매수 룰 적용 시 → 자동 경고 "33% 승률 — D+1 안착 룰 강제"
```
