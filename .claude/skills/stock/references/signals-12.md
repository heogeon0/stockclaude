## 종합 verdict 산정

`compute_signals.summary.종합` 필드:

| verdict | 매수 가중합 | 매도 가중합 |
|---|---|---|
| **강한매수** | ≥ 3.5 | 0 |
| **매수우세** | > 매도, ≤ 1.5 | < 매수 |
| **중립** | ≈ 매도 | ≈ 매수 |
| **매도우세** | < 매도 | > 매수, ≥ 1.5 |
| **강한매도** | 0 | ≥ 3.5 |

---

## 시그널 활용 룰

### 보유 종목

- **강한매수 / 매수우세** + 비중 < 25%: 피라미딩 검토 (변동성×재무 셀 룩업)
- **중립**: 홀딩, 트레일링 스탑 점검
- **매도우세 / 강한매도**: 부분 / 전량 청산 검토

### 신규 진입 후보

- **강한매수**: 진입 가능 (변동성×재무 셀 사이즈 적용)
- **매수우세**: 1차 진입 (잔여는 추가 트리거 대기)
- **중립**: 진입 보류
- **매도우세 / 강한매도**: ❌ 진입 비추

### compute_signals 서버 에러 시 우회

지표 (`compute_indicators`) + 이벤트 (`detect_events`) + 등급 (`compute_score`) 조합으로 수동 verdict 산정:

- RSI > 80 + 그랜빌 SMA60 매도 + 거래량 약화 → 매도우세
- 일목 매수 + 래리 돌파 + RSI 정상권 → 매수우세
- 등

## 단일 시그널 의존 금지

- **종합 verdict 우선** — 단일 시그널 (예: RSI 과매수) 만으로 매도 결정 X
- 매수 N 개 + 매도 1 개 → 매수우세 (단일 매도 무시 가능)
- 단, RSI 90+ 같은 극단치는 단독으로도 매도 가중치 큼

## VCP 패턴 (미너비니 SEPA 차원 부가정보)

`compute_signals.signals[2].부가정보.VCP`:

- `is_vcp: true` + `contractions: [22, 9]` (수축률) → **VCP 정석 패턴**
- `volume_trend: '감소'` → 정석 (수축 단계 적합)
- `피봇`: 돌파 트리거 가격

→ VCP 정석 + 거래량 감소 + 피봇 돌파 + 거래량 폭증 = 미너비니 진입 시그널 핵심
